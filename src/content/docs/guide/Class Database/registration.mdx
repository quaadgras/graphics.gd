In order to register a new class that will be available within the editor, create a Go struct that embeds the
`T.Extension` type of the object you would like to extend.

All struct fields beginning with an uppercase letter will be exported as properties in the editor, and all methods
beginning with an uppercase letter will be exported for use in `.gd` scripts.

```go
package main

import (
	"fmt"

	"graphics.gd/classdb"
	"graphics.gd/classdb/Node"
	"graphics.gd/startup"
)

type MyClass struct {
	Node.Extension[MyClass]

	MyProperty string // properties will be exported as snake_case
}

// Implements the Ready method of the [Node.Interface]
func (*MyClass) Ready() {
	fmt.Println("MyClass is ready!")
}

// methods will be available to scripts in snake_case, ie. my_script_method
func (*MyClass) MyScriptMethod() {
	fmt.Println("MyClass script method called!")
}

func main() {
	classdb.Register[MyClass]()
	startup.Scene()
}

```

<Aside>It is also acceptable to register your class within a packages' `init` function, so that you don't
    have to keep track of everything within `main`.</Aside>


### Struct Tags
`graphics.gd` will automatically register everything inside your struct for use with scripts, struct tags
are available for more control on how struct fields will be presented to the engine.

* `gd:"rename"` sets the name of of the property within the engine.
*  `range:"min_value, max_value, step, flags"` configures the allowed range of values for the property, this property will be represented as a slider in the editor.
* `group:"name"` sets the associated group that the property will be organised under within the editor.
* `gd:"-"` hide this field from the engine, use this if you need a field exported in Go but not in the engine.

### Run in the Editor
By default, methods and notifications will not be called when running your project inside the editor. To enable this,
embed `classdb.Tool` to your extension class.

```go
type MyClass struct {
	Node.Extension[MyClass]
	classdb.Tool // methods will run in the editor.
}
```

Use [Engine.IsEditorHint](https://pkg.go.dev/graphics.gd/classdb/Engine#IsEditorHint) to check whether you are running 
in the editor or not.

### Constructors
Go doesn't have constructors like `.gd` script, instead, if you have a function that you are using to initialize your
structures, you can register this with the engine, so that it will be used as the constructor by scripts.

```go
package main

import (
	"graphics.gd/classdb"
	"graphics.gd/classdb/Node"
	"graphics.gd/startup"
)

type MapContainer struct {
	Node.Extension[MapContainer]

	my_map map[string]string
}

func NewMapContainer() *MapContainer {
	return &MapContainer{
		my_map: make(map[string]string),
	}
}

func main() {
	classdb.Register[MyClass](NewMapContainer)
	startup.Scene()
}
```

### Static Functions
You can register additional functions to be bundled with your class and available to scripts by passing them
to `classdb.Register`. You can also rename these functions by passing a `map[string]any`.

```go
package main

import (
	"graphics.gd/classdb"
	"graphics.gd/variant/Object"
	"graphics.gd/startup"
)

type UtilityFunctions struct { Object.Extension[UtilityFunctions] }

func DoSomething() {}

func main() {
	classdb.Register[UtilityFunctions](DoSomething)
	startup.Scene()
}
```

### Renaming Functions
Functions and methods can be renamed inside the engine by passing a `map[string]any` to `classdb.Register` with the
functions and methods you would like to specify the name of.

```go
package main

import (
	"graphics.gd/classdb"
	"graphics.gd/variant/Object"
	"graphics.gd/startup"
)

type MyObject struct { Object.Extension[MyObject] }
func (*MyObject) DoSomething() {}

func main() {
	classdb.Register[RenamedFunctions](map[string]any{
		"do_something_renamed": MyObject.DoSomething,
	})
	startup.Scene()
}
```


### Constants and Enums
If you use the `graphics.gd/variant/Enum` package to define your enums, they will be
registered automatically if they are used by any functions or methods. You can also
register constants by passing them in a `map[string]int` to `classdb.Register`.

```go
package main

import (
	"graphics.gd/classdb"
	"graphics.gd/variant/Object"
	"graphics.gd/variant/Enum"
	"graphics.gd/startup"
)

const (
	MyConstant = 1
)

type MyEnum Enum.Int[struct{
	SomeValue MyEnum
}]

var MyEnums = Enum.Values[MyEnum]()

type HasEnumsConstants struct { Object.Extension[HasEnumsConstants] }
func (*HasEnumsConstants) Enum() MyEnum {
	return MyEnums.SomeValue
}

func main() {
	classdb.Register[HasEnumsConstants](map[string]int{
		"MY_CONSTANT": MyConstant,
	})
	startup.Scene()
}
````

### Singletons
You can register a class as a singleton by embedding the `Singleton[T]` type instead of
`Extension[T]`. This will intialize a singlet instance of the object on startup and make
it available to scripts as a global variable with the same name as the struct (or the `gd`tag).

Singleton pointers will also be automatically filled in when other `Extension[T]` types refer
to them as a field. If the singleton is also a `Node.Any`, then it will be added to the scene
tree on startup as well.

```go
package main

import (
	"fmt"

	"graphics.gd/classdb"
	"graphics.gd/classdb/Engine"
	"graphics.gd/classdb/Node"
	"graphics.gd/classdb/Node3D"
	"graphics.gd/classdb/SceneTree"
	"graphics.gd/startup"
)

type Singleton struct {
	Node.Singleton[Singleton]
}

func (g *Singleton) Print() {
	Engine.Println("Hello, graphics.gd!")
}

type Game struct {
	Node3D.Extension[Game]

	Singleton *Singleton
}

func (g *Game) Ready() {
	g.Singleton.Print()
	fmt.Println("singleton in scene tree? ", g.Singleton.AsNode().IsInsideTree()) // true
}

func main() {
	classdb.Register[Singleton]()
	classdb.Register[Game]()
	startup.LoadingScene()
	SceneTree.Add(new(Game))
	startup.Scene()
}
```

### Inheritance

Go doesn't have inheritance or OOP built into the language but you can achieve
a similar effect of sharing common functionality between extension classes by using 
struct embedding.

```go
package main

import (
  "graphics.gd/classdb"
  "graphics.gd/classdb/Node"
  "graphics.gd/startup"
)

type Common[T classdb.Class] struct {
  Node.Extension[T]
}

func (c *Common[T]) Ready() {
  fmt.Println("Ready from Common!")
}

type MyClass struct {
  Common[MyClass]
}

type MyOtherClass struct {
  Common[MyOtherClass]
}

func main() {
	classdb.Register[MyClass]()
	classdb.Register[MyOtherClass]()
	startup.Scene()
}

```

If you *really* want to have an inheritance-hierarchy in the engine, use the advanced 
[classdb.ExtensionInherits](https://pkg.go.dev/graphics.gd/classdb#ExtensionInherits)
technique (fields and methods will not be promoted in Go, you have to use Super() to
access the parent):

```go
package main

import (
  "graphics.gd/classdb"
  "graphics.gd/classdb/Node"
  "graphics.gd/startup"
)

type Parent struct {
  Node.Extension[Parent]
}

type Child struct {
  classdb.ExtensionInherits[Parent, Child]
}

func main() {
	classdb.Register[Parent]()
  classdb.Register[Child]()
  startup.Scene()
}
```
